# SQL Analysis Formatter 利用ガイド

T-SQLの識別子を和名へ変換し、Excel上にSQL解析結果と定義表を作成するツールです。

## 動作環境

- 64ビット版Windows
- VBAマクロを利用できるデスクトップ版Excel
- `SqlAnalysisFormatter.Parser.exe`を実行できる環境

parserは.NET 8の実行環境を内包した単一EXEです。通常は.NETを別途インストールする必要はありません。

## 受け取るファイル

配布方法によって、次のどちらかを受け取ります。

| 配布方法 | 使用するファイル |
| --- | --- |
| セットアップ済みブック | `SqlAnalysisFormatter.xlsm`、`SqlAnalysisFormatter.Parser.exe` |
| 既存ブックへ導入 | `SqlAnalysisFormatter.bas`、`SqlAnalysisFormatter.Parser.exe`、`README.md` |

## 最短で使う

`SqlAnalysisFormatter.xlsm`を受け取った場合は、この方法が最短です。

1. 作業用フォルダを作成します。
2. `SqlAnalysisFormatter.xlsm`と`SqlAnalysisFormatter.Parser.exe`を同じフォルダへ置きます。
3. ダウンロードしたファイルがWindowsにブロックされている場合は、各ファイルの`プロパティ`を開き、`許可する`（ブロックの解除）を選択します。項目が表示されなければ、この操作は不要です。
4. `SqlAnalysisFormatter.xlsm`をExcelで開きます。
5. セキュリティ警告が表示された場合は、配布元を確認したうえでマクロを有効化します。
6. `変換定義`シートへ定義を入力し、`SQL解析`シートで`解析`ボタンを押します。

推奨する配置は次のとおりです。

```text
SqlAnalysisFormatter/
|-- SqlAnalysisFormatter.xlsm
`-- SqlAnalysisFormatter.Parser.exe
```

会社のセキュリティポリシーでマクロやEXEが禁止されている場合は、設定を回避せず管理者へ確認してください。

## 初回セットアップ

既存のExcelブックへ`.bas`を導入する場合の手順です。

1. 対象ブックのバックアップを作成します。
2. 対象ブックをExcelで開き、`Excelマクロ有効ブック (*.xlsm)`として保存します。
3. `Alt + F11`でVBAエディターを開きます。
4. `ファイル` -> `ファイルのインポート`を選択します。
5. `SqlAnalysisFormatter.bas`を選択して取り込みます。
6. VBAエディターを閉じ、Excelで`Alt + F8`を押します。
7. マクロ一覧から`SetupWorkbook`を選び、`実行`を押します。
8. `変換定義`、`SQL解析`、`アウトプット`の3シートと、`解析`、`クリア`、`コピー`の3ボタンが作成されたことを確認します。
9. ブックを保存します。
10. 保存したブックと同じフォルダへ`SqlAnalysisFormatter.Parser.exe`を置きます。

利用者がインポートするのは`SqlAnalysisFormatter.bas`だけです。`SqlAnalysisFormatterTests.bas`は開発者用なので取り込みません。

### Bootstrapから展開する場合

配布元からユーザー向けBootstrapを受け取った場合は、PowerShellで次のように実行します。実行中にGitHubへの通信は行いません。

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\SqlAnalysisFormatter.user.bootstrap.ps1 -TargetDirectory C:\Tools\SqlAnalysisFormatter
```

展開先には`SqlAnalysisFormatter.bas`、`SqlAnalysisFormatter.Parser.exe`、`README.md`が作成されます。展開後は上記の「初回セットアップ」に進みます。

## 使い方

### 1. 変換定義を入力する

`変換定義`シートの2行目以降へ入力します。

| A: 所属テーブルID | B: 所属テーブル和名 | C: フィールドID | D: フィールド和名 |
| --- | --- | --- | --- |
| users | ユーザー | user_id | ユーザーID |
| users | ユーザー | name | 氏名 |
| - |  | status | 状態 |

- `users.user_id`は`users.ユーザーID`へ変換します。テーブルID自体は変換しません。
- A列が`-`の場合、`status`のような単独フィールドも`状態`へ変換します。
- B列のテーブル和名は、アウトプットの参照テーブル表示に使用します。
- `#temp_users`のような一時テーブルは、A列へ`#`を含む名前を入力します。SQLで別名を付けても、アウトプットの参照テーブル和名を物理名から解決します。
- D列が空欄または`(和名未取得)`の定義は変換しません。
- `AS`直後の単独IDは列エイリアスとして扱い、変換しません。

### 2. SQLを貼り付ける

1. A5:SQL Mk-2（A5M2）でT-SQLを開きます。
2. `Ctrl + Q`で整形します。
3. 整形結果全体をコピーします。
4. `SQL解析`シートのA2セルを選択し、通常どおり貼り付けます。

改行されたSQLは、A2、A3、A4のように1行ずつ別セルへ貼り付けられます。SQL全体をA2の1セルへ押し込む必要はありません。A列2行目以降の空白でない行を、1つのクエリとして解析します。

貼り付け後は、例えば次のように並びます。

| セル | 内容 |
| --- | --- |
| A2 | `SELECT` |
| A3 | `    users.user_id,` |
| A4 | `    users.name` |
| A5 | `FROM users` |

解析後はB3が`users.ユーザーID,`、B4が`users.氏名`となり、変換内容は同じ行のC列以降へ表示されます。

### 3. 解析する

`SQL解析`シートの`解析`ボタンを押します。

| 確認場所 | 出力内容 |
| --- | --- |
| SQL解析 B列 | 行ごとの和名変換後SQL |
| SQL解析 C列以降 | その行で変換された内容を1セルずつ表示 |
| アウトプット | AST解析から作成した定義表 |

## アウトプット

- SELECT全体は`＜DB入出力項目定義＞`形式で出力します。
- サブクエリは同じ表形式で、A1を`サブクエリ[名前]`として出力します。
- INSERT SELECT、DELETE、UPDATEは`＜データ移送表＞`形式で出力します。
- WITH句は名前付きサブクエリとして扱います。
- ネストしたサブクエリは内側から外側、最後にクエリ全体の順で出力します。
- 無名サブクエリには`SQ1`、`SQ2`の名前を付けます。
- 未対応のSQLは、SQL解析シートのB列と同じ和名変換後SQLをA列へ1行ずつ出力します。
- フォールバック時は1行空けた末尾へ`フォールバック原因`と、原因クエリがあるアウトプットシートの行範囲を表示します。
- 更新系では、テーブル列を参照する式を`移送元`へ、変数・定数・テーブル列を参照しない関数を`移送方法ほか`へ出力します。

アウトプットはA列からCL列を使い、MSゴシック9ポイント、列幅1.14、行高13.5、目盛り線なしで整形します。

### 成果物をコピーする

`アウトプット`シートの`コピー`ボタンを押すと、A1からCL列の最終出力行までをExcelのクリップボードへ登録します。貼り付け先では通常の`Ctrl + V`を使用します。

## クリアするときの注意

`クリア`ボタンを押すと確認ダイアログを表示します。承認すると、次の内容をクリアします。

- `変換定義`シートの2行目以降
- `SQL解析`シートの2行目以降
- `アウトプット`シートの内容と出力書式

1行目のヘッダーが変更されていても復元します。変換定義も削除されるため、再利用する定義は事前に別ファイルへ保管してください。

## バージョンを更新する

1. 対象ブックのバックアップを作成します。
2. `Alt + F11`でVBAエディターを開きます。
3. `標準モジュール`にある既存の`SqlAnalysisFormatter`を右クリックし、`SqlAnalysisFormatterの解放`を選びます。
4. エクスポート確認では、別途バックアップ済みなら`いいえ`を選択します。
5. 新しい`SqlAnalysisFormatter.bas`をインポートします。
6. `SetupWorkbook`を実行し、ブックを保存します。
7. `SqlAnalysisFormatter.Parser.exe`も新しいファイルへ置き換えます。

既存の入力データはモジュールの更新だけでは削除されません。

## 困ったとき

| 症状 | 確認すること |
| --- | --- |
| シートやボタンがない | `Alt + F8`から`SetupWorkbook`を実行します。 |
| マクロを実行できない | ファイルのブロック解除、Excelのマクロ警告、組織のセキュリティポリシーを確認します。 |
| B列が空のまま | `変換定義`のA列、C列、D列と、`SQL解析`のA2以降を確認します。 |
| フィールドが変換されない | テーブル修飾付きならA列とC列、単独フィールドならA列が`-`か確認します。 |
| 列エイリアスが変換されない | `AS`直後は意図的に変換対象外です。 |
| アウトプットにSQLがそのまま出る | 末尾の`フォールバック原因`を確認します。parser未配置ならEXE名、配置場所、Windowsのブロック状態を確認します。未対応構文なら原因を添えて開発者へ連絡します。 |
| 参照テーブルが`(和名未取得)`になる | 対象テーブルIDに対応するB列の和名を確認します。CTE名など変換定義にない名前は`(和名未取得)`になります。 |

parserが見つからない場合や実行に失敗した場合も、B列とC列以降の和名変換は継続します。この場合、アウトプットは分解せず、和名変換後SQLをA列へ1行ずつ表示し、末尾へ原因を表示します。

## セキュリティと通信

- parserの解析処理はPC内で完結します。
- 利用時にGitHubへ通信しません。
- 一時ファイルはWindowsの一時フォルダへ作成し、処理後に削除します。
- 配布元が確認できないマクロやEXEは実行しないでください。
